# План: только блок 3 на index, остальное — отдельные страницы

## Принцип

- **index.html** — не перегружаем. Добавляется **только блок 3** «Вам это подходит, если…» с двумя карточками и кнопками перехода на отдельные страницы. Меню главной и остальной контент index не меняем (без добавления пунктов «Для собственников» / «Для ИТ-лидеров» в шапку главной).
- **Страницы owners, for-leaders, risks, ask-ai** — отдельные документы, открываются по кнопкам из блока 3 и по ссылкам друг из друга. К главной странице не относятся; весь их контент живёт только на этих страницах.

## Исходные данные

- Концепция: [концепция_продвижения.txt](концепция_продвижения.txt) (строки 60–381).

- Стили: [styles.css](styles.css) — общие переменные и классы.

## 2. Навигация между страницами

- **С главной:** Пользователь попадает на owners/ или for-leaders/ только по кнопкам в блоке 3.
- **На отдельных страницах (owners, for-leaders, risks, ask-ai):** У каждой свой header: логотип ведёт на главную (`../index.html` или `../`), в меню — пункты «Для собственников», «Для ИТ-лидеров», «Риски», «Спросите ИИ», «На главную»/«Роли и форматы» (на главную), «Контакты» (главная#contacts). Так пользователь может переходить между целевыми страницами и возвращаться на index без перегрузки главной.
- **Связанные разделы** на страницах owners и for-leaders: блок с ссылками «Узнать о рисках» → `../risks/`, «Проверить ИИ» → `../ask-ai/`, «Роли и форматы» → `../#formats`.

## 3. Отдельная страница «Для собственников и советов директоров» (/owners)

**URL:** `owners/index.html` — отдельная страница, не часть index. Открывается по кнопке «Оценить риски» из блока 3.

**Структура (по концепции строки 145–223):**

1. Шаблон: свой `<header>` (логотип → главная, меню: Для собственников, Для ИТ-лидеров, Риски, Спросите ИИ, Контакты), `<main>`, `<footer>`, кнопка «Наверх». Подключение `../styles.css`.
2. Заголовок секции: «Вы принимаете решения по ИТ».
3. Подзаголовок: текст про «чёрный ящик» (из концепции).
4. Блок «Ситуации» — 4 карточки/блока с заголовком, текстом и иконкой (иконки — символы или простые CSS-псевдоэлементы, без новых изображений): планируете оптимизацию; уже провели сокращения; перед сделкой M&A; совет спрашивает про готовность.
5. Акцентный блок «Моя роль»: одна цитата/выделенный текст.
6. «Что вы получите» — 3 блока: Прозрачность, Обоснованные решения, Спокойствие (заголовок + текст + иконка).
7. «Как это работает» — 3 шага: Аудит (2–4 недели), Презентация совету, Ваш выбор (варианты А–Г).
8. «Связанные разделы» — три карточки/кнопки: Узнать о рисках → `../risks/`, Проверить ИИ → `../ask-ai/`, Посмотреть форматы → `../#formats`.
9. Нижний CTA: «Готовы увидеть риски в рублях?», текст про 30 мин, кнопки «Записаться на консультацию» → `../#contacts` и «Скачать чек-лист» → `../resources/checklist.pdf` (или существующий checklist.html).

**Стиль:** Только существующие классы и переменные: `.section`, `.section-alt`, `.section-light`, `.container`, `.section-title`, карточки в духе `.format-card`, `.btn`, callout в духе `.about-callout`. Новые классы только там, где нужна специфичная сетка (например `.steps-list`, `.benefits-grid`).

## 7. Единый цвет и дизайн (только для отдельных страниц)

- **Стили:** Страницы owners, for-leaders, risks, ask-ai подключают один [styles.css](styles.css) (`../styles.css`). Цвета и типографика — те же, что на главной (переменные `:root`).

- **Компоненты:** Повторяющиеся блоки (шаги, ситуации) — переиспользуемые классы в [styles.css](styles.css) в рамках существующих переменных.
- **Иконки:** Unicode или CSS-псевдоэлементы, без новых изображений.
